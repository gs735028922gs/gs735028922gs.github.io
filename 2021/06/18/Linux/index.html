Linux

一、linux文件目录结构

—==linux世界里，一切皆文件==

  1. ==/bin==：Binary的缩写，存放最经常用的命令
  2. /sbin：superuser Binary，存放系统管理员使用的系统管理程序
  3. ==/home==：存放普通用户的主目录，每个用户都有一个自己的目录
  4. ==/root==：系统管理员（超级权限者）的用户主目录
  5. /lib：系统开机所需的最基本的动态连接共享库
  6. /lost+found：一般情况下是空的，非法关机后就存放一些文件
  7. ==/etc==：所有系统管理所需要的配置文件和子目录，比如安装mysql的my.conf
  8. ==/usr==：==极为重要==。用户的很多应用程序和文件都放在该目录下，类似windows的program files
  9. /boot：存放启动linux时使用的一些核心文件
  10. /proc：虚拟的目录，系统内存的映射，访问这个目录来获取系统信息
  11. /srv：service，存放一些服务启动后需要提取的数据
  12. /sys：安装了2.6内核中出现的一个文件系统sysfs
  13. /tmp：存放临时文件
  14. /dev：类似window设备管理器，所有硬件用文件的形式存储
  15. ==/media==：linux自动识别设备如U盘，光盘，识别后把识别的设备挂载在这个目录下
  16. /==mnt==：让用户临时挂载别的文件系统
  17. /opt：给主机额外安装软件所存放的目录，把安装文件放在这个目录下
  18. ==/usr/local==：另一个给主机额外安装软件的目录，一般是通过编译源码的方式安装程序
  19. ==/var==：存放在不断扩充的东西，习惯将经常修改的目录放在这个目录下，包括日志文件
  20. /selinux：SELinux是一种安全子系统，能控制程序只能访问特定文件，有三种工作模式

6.权限

ls -l显示的内容如下：

前0-9位：

  • 第0位：确定文件类型（d，-，l，c，b）

    • l是连接，即快捷方式
    • d是目录，即文件夹
    • c是字符设备文件，鼠标，键盘
    • b是块设备，硬盘
    • -代表它是普通文件

  • 第1-3位确定==文件所有者==拥有该文件的权限

  • 第4-6位确定==与文件拥有者同一组的用户==拥有该文件的权限

  • 第7-9位确定==其他用户==拥有该文件的权限

  • 紧跟后面的数字：文件：硬连接数；目录：子目录数。

==rwx权限详解：==

  • rwx作用于文件：

    • r：可以读取，查看
    • w：可以修改，但不一定可以删除。删除的前提是对该文件所在的目录有写的权限
    • x：可执行

  • rwx作用于目录：==——目录就是目录，不包括内部的文件==

    • r：可以读取，ls查看内容
    • w：可以修改，对目录内文件进行：创建、删除、重命名
    • x：可以进入该目录

    对于目录来说：

    • 仅拥有x权限，可以进入该目录，而不能使用ls命令，但如果拥有该目录内部文件的相关权限，也可以对文件执行操作。

  • rwx也可以用数字表示：r=4，w=2，x=1。

==修改权限：==

使用==chmod指令，修改文件或目录的权限==

  • 第一种方式：+，-，=变更权限

    u：所有者；g：所在组；o：其他人；a：所有人（u，g，o的总和）

    chmod u=rwx,g=rx,o=x 文件/目录名

    chmod o+w 文件/目录名

    chmod a-x 文件/目录名

  • 第二种方式：使用数字变更权限

    rwx也可以用数字表示：r=4，w=2，x=1。

    chmod u=rwx,g=rx,o=x 文件目录名

    相当于：

    chmod 751 文件目录名

修改文件所有者：

​ 改变所有者：chown 用户名 文件/目录

​ 改变所有者和所在组：chown 用户名:组名 文件/目录

修改文件/目录所在组：

​ 改变所在组：chgrp 组名 文件/目录

八、定时调度

1.crond任务调度

crontab 指令 进行定时任务的设置

​ 任务调度：系统在某个时间执行特定的命令或程序，可以定时调用脚本或指令

​ 任务调度分类：

    1. 系统工作：一些重要的工作需要周而复始地执行，如病毒扫描
    2. 个别用户工作：个别用户希望执行的工作

基本语法：

​ ==crontab [选项]==

​ 选项：

           1. -e：编辑crontab定时任务---edit
           2. -l：查询crontab任务--list
           3. -r：删除当前用户的所有crontab任务--remove

​ 特殊：

​ 重新启动任务调度：service crond restart

示例：每分钟执行(ls -l /etc/ > /tmp/to.txt)命令

==时间规则：==

​ 细节参数：

      1. 第一个* :一小时中的第几分钟 (0~59)
      2. 第二个* :一天的第几个小时 (0~23)
      3. 第三个* :一月中的第几天 (1~31)
      4. 第四个* :一年中的第几月 (1~12)
      5. 第五个* :一周的星期几 (0~7)--0和7都是星期天
      6. 分，时，日，月，星期几

​ 特殊符号：

  1. “ * “:代表“每一个”时间，每分钟、每小时、每个月…….
  2. “ , “:代表不连续的时间—“0 8,12,16 * * *”—–代表每天的8点、12点、16点都执行一次
  3. “ - “：代表连续的时间范围—-“0 5 * * 1-6”—-代表周一到周六凌晨五点执行
  4. “ */n “:代表每隔多久执行一次—–” */10 * * * * “——代表每隔十分钟执行一次

十一、进程管理（究极重点）

  • linux每个执行的程序都称为一个进程，每个进程都分配一个ID号（pid）
  • 每个进程都可以以两种方式存在：前台、后台。前台进程就是用户目前屏幕上可以进行操作的。后台进程是实际在操作，屏幕上无法看到的进程，通常使用后台方式执行。
  • 一般系统的服务都是后台进程，且都会常驻在系统中，直到关机才结束

==1.显示系统执行的进程==

==ps命令：==

​ ps -a：显示当前终端的所有进程信息
​ ps -u：以用户的格式显示进程信息

​ ps -x：显示后台进程运行的参数

​ ps -e：显示所有进程

​ ps -f：全格式

​ ps -ef：以全格式显示当前的所有进程

​ 一般组合使用：ps -aux

ps详解：

​ 1.查看某个进程： ps -aux | grep xxx

​ 2.显示的信息选项：

  • System V 展示风格
  • USER：用户名称
  • ==%CPU：占用CPU百分比==
  • ==%MEM：占用物理内存百分比==
  • ==VSZ：占用虚拟内存大小（KB）==
  • ==RSS：占用物理内存大小（KB）==
  • TTY：终端名称
  • STAT：进程状态
    • S：睡眠
    • s：是会话的先导进程
    • N：标识进程拥有比普通优先级更低的优先级
    • R：正在运行
    • D：短期等待
    • Z：僵死进程
    • T：被跟踪或停止等
  • STARTED：进程启动时间
  • TIME：进程使用CPU的总时间
  • COMMAND：进程所用的命令和参数

通过进程名杀死进程，也支持通配符：killall 进程名 （同时杀死其子进程）

==4.服务管理==

服务的本质就是进程，是运行在后台的。服务通常会监听某个端口，等待其他程序的请求，所以又称为守护进程。

service管理指令：

  1. service 服务名 [start|stop|restart|reload|status]
  2. CentOS7.0以后，很多服务不再使用service，而是systemctl
  3. service指令管理的服务在 /etc/init.d查看

查看服务名：

  1. 看到全部：setup
  2. 看到service指令管理的服务：ls -l /etc/init.d

服务的运行级别：——前面讲过

  • 级别0：系统停机状态
  • 级别1：单用户工作状态，root权限，用于系统维护，不能远程登陆
  • 级别2：多用户状态，没有NFS，不支持网络
  • ==级别3：==完全的多用户状态，有NFS，无界面，登陆后进入控制台命令模式
  • 级别4：保留
  • ==级别5：==X11控制台，登陆后进入GUI图形模式
  • 级别6：系统正常关闭并重启

chkconfig指令：

  1. 可以给服务的各个运行级别设置自 自动/关闭

  2. chkconfig指令管理的服务在 /etc/init.d查看

  3. centOS7.0后，很多服务用systemctl管理

    基本语法：

    查看服务：chkconfig –list [|grep xxx]

    chkconfig 服务名 –list

    chkconfig –level 5 服务名 on/off

==systemctl管理指令：==

  1. 基本语法：systemctl [start|stop|restart|status] 服务名
  2. systemctl指令管理的服务在 /user/lib/systemd/system 查看

systemctl设置服务的自启动状态

  1. 查看服务开启启动状态：systemctl list-unit-files [|grep 服务名]
  2. 设置服务开机启动：systemctl enable 服务名
  3. 关闭服务开机启动：systemctl disable 服务名
  4. 查询某个服务是否是自启动的：systemctl is-enabled 服务名

防火墙打开或关闭指定接口–firewall指令

  1. ==打开端口：firewall-cmd –permanent –add-port=端口号/协议==

  2. ==关闭端口：firewall-cmd –permanent –remove-port=端口号/协议==

  3. ==重新才入，才能生效：firewall-cmd –reload==

  4. ==查询端口是否开放：firewall-cmd –query-port=端口/协议==

查看端口的协议：netstat -anp | more

​ -p：通过指定监控进程id来仅仅监控某个进程的状态

M：以内存使用率排序–%MEM

N：以PID排序

q：退出top

十二、RPM和YUM

1. RPM包的管理

RedHat Package Manager，（RedHat软件包管理工具）RPM用于互联网下载包的打包及安装工具，类似windows的setup.exe，它生成“.RPM”扩展名文件。

RPM包查询指令：
查询已安装的RPM列表：rpm -qa | grep xxx

RPM报名基本格式：
firefox-60.2.2-1.el7.centos.x86_64

​ 名称：firefox

​ 版本号:-60.2.2-1

​ 使用操作系统：el7.centos.x86_64（32位：i686,i386，通用：noarch）

RPM包其他指令：

  • 查询安装的所有RPM软件包：rpm -qa
  • 查询软件包是否安装：rpm -q firefox
  • 查询软件包信息：rpm -qi 软件包名
  • 查询软件包中的文件：rpm -ql
  • 查询文件所属的软件包：rpm -qf 文件全路径名

RPM包的卸载：

​ 删除软件包：rpm -e RPM包的名称

​ 如果其他软件包依赖要删除的软件包，会产生错误信息：

​ removing these packages would break dependencies:……..

​ 想要强制删除：rpm -e –nodeps (谨慎使用)

RPM包的安装：
语法：

​ rpm -ivh RPM包全路径名称

参数：

-i：install 安装

-v：verbose 提示

-h：hash 进度条

2.YUM

YUM使一个shell前端软件包管理器。基于RPM包管理，能够从指定的服务器自动下载RPM包并按钻过，可以自动处理依赖性关系，并且一次安装所有依赖的软件包

基本指令：
查询Yum服务器是否有需要安装的软件：

​ yum list | grep xxx

安装指定的yum包：

​ yum install xxx

其他指令

clear：清屏

ps -ef | grep atd：检测atd是否在运行

ifconfig：查询网络信息
